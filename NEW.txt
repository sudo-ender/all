3.	Какие основные подходы используются для локального хранения данных в мобильных приложениях? Опишите сценарии их применения. 
4.	Перечислите и кратко охарактеризуйте основные принципы проектирования пользовательского интерфейса (UI) и пользовательского опыта (UX) для мобильных приложений. 
5.	Каким образом мобильные приложения взаимодействуют с внешними сервисами и API? Опишите основные концепции и механизмы.
6.	Что такое мобильное приложение и какие типы приложений существуют?
7.	Основные этапы разработки мобильного приложения?

13.	Как организовать асинхронную работу и многопоточность в мобильных приложениях? Какие инструменты и лучшие практики применять? 
14.	Какие варианты локального хранения данных доступны и когда их применять? 
15.	Как мобильное приложение взаимодействует с внешними сервисами и какие меры безопасности при этом необходимы? 
16.	Какие основные методы оптимизации производительности и отзывчивости мобильного приложения?

=========================================
=

1. **Нативное vs кроссплатформа** — нативное: производительность + полный доступ, но дорого; кроссплатформа: один код, быстрее, но хуже производительность.

2. **Жизненный цикл Activity** — `onCreate` (создание), `onStart` (видимость), `onResume` (фокус), `onPause` (потеря фокуса), `onStop` (скрыто), `onDestroy` (уничтожение). + `onSaveInstanceState` для сохранения состояния.

3. **Локальное хранение** — SharedPreferences (настройки), SQLite/Realm (структура), файлы (медиа), Keychain/Keystore (секреты).

4. **Принципы UI/UX** — простота, доступность, обратная связь, единообразие, адаптивность, удобство навигации.

5. **Взаимодействие с API** — HTTP (CRUD)-запросы (REST/GraphQL), JSON, токены, асинхронность, обработка ошибок.

6. **Типы приложений** — нативные, гибридные, кроссплатформенные (Flutter/RN), PWA.

7. **Этапы разработки** — идея → анализ → дизайн → разработка → тестирование → релиз → поддержка.

8. **Жизненный цикл Activity и его значение** — управление состояниями экрана для сохранения данных и ресурсов.

9. **REST API** — стиль обмена данными по HTTP, источник данных для приложения.

10. **Принципы UX** — юзабилити, предсказуемость, минимализм, обратная связь.

11. **Меры безопасности** — HTTPS, шифрование, токены в Keychain, SSL-pinning, валидация.

12. **Архитектурные шаблоны** — MVC (простой, но сложный контроллер)(Модель-Представление-Контроллер), MVP (лучше тестируемость)(Модель-Представление-Презентер), MVVM (биндинг ( обновляется сам при изм. данных) , реактивность)(Модель-Представление-МодельПредставления).

13. **Асинхронность** — Coroutines/RxJava/GCD, не блокировать UI, пулы потоков, отмена задач.

14. **Локальное хранение** (повтор вопроса №3) — см. пункт 3.

15. **Взаимодействие с сервисами + безопасность** — HTTP-клиенты, HTTPS, SSL-pinning, токены в заголовках.

16. **Оптимизация производительности** — кеширование, сжатие изображений, ленивая загрузка, профилирование, избегание утечек.
